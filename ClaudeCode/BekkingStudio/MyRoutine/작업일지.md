## 2026-02-23 (Day 2) - 버그 수정

### 수행 작업
1. **자정 후 새 기록 차단 버그 수정** (`3f85ee2`)
   - **증상**: 마감 후 자정이 지나도 "이미 마감된 날짜입니다" 에러 발생
   - **원인**: `currentDate`가 앱 로드 시에만 설정되고, 자정 이후 갱신되지 않음
   - **수정**: `updateClock()`에서 매초 `getTodayStr()`와 비교하여 날짜 변경 시 자동 전환

2. **중복 활동명 원하는 삶 체크 오류 수정** (`3a7c36c`)
   - **증상**: 같은 활동을 여러 번 기록 시, 원하는 삶 체크가 첫 번째만 토글됨 → 캘린더 별표 미표시
   - **원인**: `renderDesiredList`에서 `dd.entries.find(e => e.activity === a.activity)` 사용 → 항상 첫 번째 매치만 반환
   - **수정**: `dd.entries.find(e => e.id === a.id)` + `a.id`로 변경하여 각 항목 개별 토글 가능

### 핵심 발견사항
- 사용자가 하루 사용 후 자정 기준 초기화가 안 되는 치명적 버그 발견
- 중복 활동명이 있을 경우 원하는 삶 퍼센트가 실제보다 낮게 계산되는 버그 발견

3. **자정 자동 마감 기능** (`d4b3964`)
   - 자정이 지나면 기록 중인 날짜 자동 마감 (status → completed, closedAt → 23:59)
   - 미분류 활동 자동 분류 (프리셋 설정 기반)
   - 설정 섹션 삭제 (불필요한 UI 정리)

4. **과거 날짜 분석 탭에서 분류/원하는 삶 수정 기능** (`e5cac56`)
   - **캘린더 → 분석 탭 이동**: 과거 완료 날짜 클릭 시 분석 탭으로 바로 이동
   - **과거 날짜 편집 배너**: coral 그라데이션 배너에 "📅 {날짜} 분석 수정 중" + "오늘로 돌아가기" 버튼 표시
   - **바 차트 분류 토글**: 각 활동 옆에 고정/유동 토글 버튼 추가 (수면 제외), `setCategoryInAnalysis()`로 같은 활동명 엔트리 일괄 변경
   - **저장 완료 버튼**: 하단에 "저장 완료 · 오늘로 돌아가기" 버튼 (과거 날짜 편집 시만 표시)
   - **캘린더 0h 강조**: 원하는 삶 0h인 날짜를 주황색(coral)으로 강조 → 수정 필요 날짜 시각적 구분
   - **자정 자동 전환 방지**: `viewingPastDate` 플래그로 과거 날짜 편집 중 currentDate 강제 전환 차단
   - **i18n**: `analysis.editingDate`, `analysis.backToToday`, `analysis.saveAndReturn` 키 추가 (ko/ja/en)
   - Service Worker 캐시 버전: `myroutine-v9`
