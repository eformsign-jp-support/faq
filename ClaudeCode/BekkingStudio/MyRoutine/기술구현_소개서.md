## 2. 앱 아키텍처

### Single-File PWA (Progressive Web App)
- **파일 구조**: `MyRoutine.html` 단일 파일 (HTML + CSS + JavaScript 약 5,500줄)
- **프레임워크 없음**: 순수 Vanilla HTML/CSS/JavaScript로 구현
- **이유**: 외부 의존성 최소화, 빠른 로딩, 오프라인 동작 보장

```
MyRoutine/
├── MyRoutine.html    ← 앱 전체 (= index.html)
├── sw.js             ← Service Worker
├── manifest.json     ← PWA 매니페스트
├── icon-192.png      ← 앱 아이콘
└── icon-512.png      ← 앱 아이콘 (고해상도)
```

## 4. 주요 기능별 기술 상세

### 4-1. 활동 기록 시스템
- **데이터 구조**: `data.dates[YYYY-MM-DD].entries[]` — 시간순 활동 배열
- **프리셋 시스템**: `{ name, icon, category, color }` — 14개 기본 + 사용자 커스텀
- **기상/취침 감지**: `isWakeActivity()`, `isBedActivity()` — 다국어 프리셋명 매칭
- **수면 계산**: 취침→기상 페어링으로 자동 산출 (`calcSleepMin()`)

### 4-2. 분류 & 분석
- **고정/유동 분류**: 각 활동을 "고정 활동"(수면, 식사 등)과 "유동 활동"(운동, 독서 등)으로 분류
- **원하는 삶 체크**: 유동 활동 중 "원하는 삶"에 해당하는 활동 체크 → 달성률 계산
- **바 차트**: Chart.js 가로 바 차트로 활동별 시간 시각화
- **캘린더**: 월별 캘린더에 원하는 삶 시간 + 달성률 표시

### 4-3. 통계 대시보드
- **기간 선택**: 최근 7일 / 30일 / 전체
- **트렌드 차트**: Chart.js 라인 차트 — 활동 시간 추이, 달성률 추이
- **랭킹 카드**: 활동 TOP 5 (고정 활동 제외 필터)
- **PDF 내보내기**: html2pdf.js로 통계 전체를 PDF 파일로 저장

### 4-4. 클라우드 동기화
- **인증**: Firebase Auth + Google OAuth 2.0
- **저장소**: Firestore — 사용자별 문서 (`users/{uid}`)
- **동기화 전략**: 3초 디바운스 자동 저장, 첫 로그인 시 업로드/다운로드 선택
- **오프라인 우선**: localStorage가 기본, Firestore는 옵션 백업

### 4-5. 소셜 공유
- **이미지 카드**: HTML로 카드 렌더링 → html2canvas로 1080x1080 PNG 변환
- **공유 방식**: Web Share API (모바일) / 다운로드 (데스크톱)
- **카드 종류**: 하루 분석 카드 (도넛 차트) + 통계 카드 (랭킹)

### 4-6. 다국어 지원 (i18n)
- **지원 언어**: 한국어, 일본어, 영어
- **감지 방식**: `navigator.language` 자동 감지 (ko/ja/en, 기본 ko)
- **번역 키**: 약 120개 이상의 `t('key')` 함수 호출
- **프리셋 번역**: `PRESET_TRANSLATIONS` 객체로 프리셋명도 다국어 지원

### 4-7. 온보딩 투어
- **7단계 스텝 투어**: 환영 → 빠른기록 → 분류 → 분석 → 통계 → 하루마감 → 완료
- **하이라이트**: 타겟 요소 주변에 오버레이 + 툴팁 표시
- **재실행**: 앱 정보 모달에서 "사용법 보기" 버튼으로 다시 보기
